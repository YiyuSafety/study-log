# Web日志访问基础
## 1.日志长什么样子
192.168.1.10 - - [25/May/2026:14:30:21 +0800] "GET /admin HTTP/1.1" 404 532
访问者IP 远程登录名 认证用户名 [时间] "请求方法 访问路径 访问协议" 状态码 响应大小

## 2.请求方法
|方法  |含义    |安全分析时怎么看|
|------|--------|--------------|
|`GET`  |获取页面或资源 |最常见 |
|`POST` |提交数据   |登录、表单、接口常用   |
|`PUT`  |上传或修改资源 |普通网站少见，出现要注意   |
|`DELETE`   |删除资源   |出现要注意 |
|`HEAD` |只请求响应头   |扫描器可能会用 |

## 3.状态码
| 状态码       | 含义    | 安全分析价值         |
| --------- | ----- | -------------- |
| `200`     | 请求成功  | 访问成功           |
| `301/302` | 跳转    | 登录跳转、页面重定向     |
| `403`     | 禁止访问  | 可能访问了敏感目录      |
| `404`     | 不存在   | 大量 404 可能是目录扫描 |
| `500`     | 服务器错误 | 可能触发了后端异常      |
| `502/503` | 服务异常  | 服务不稳定或后端故障     |

## 4.敏感路径
| 路径            | 风险           |
| ------------- | ------------ |
| `/admin`      | 后台入口         |
| `/phpmyadmin` | 数据库管理入口      |
| `/.git`       | 可能泄露源码       |
| `/.env`       | 可能泄露密钥、数据库密码 |
| `/backup.zip` | 可能泄露网站备份     |
| `/upload`     | 可能和文件上传漏洞有关  |
| `/shell.php`  | 可能是 WebShell |
